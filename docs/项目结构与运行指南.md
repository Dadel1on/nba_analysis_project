# 项目结构与运行指南

本文档说明当前仓库的真实目录分工、运行入口和数据流向，方便后续维护时快速定位文件，不影响任何运行逻辑。

## 1. 目录分工

- [front](../front)：前端应用，负责页面、路由、图表和接口联调
- [backend-spring](../backend-spring)：后端应用，负责 REST API、数据库访问和 Spark 任务编排
- [docs](.)：项目级说明文档、集群启动指南和数据处理说明

## 2. 前端结构

前端目录中最重要的部分是：

- `src/api`：请求封装和接口类型
- `src/pages`：页面级入口
- `src/views`：业务视图组件
- `src/router`：路由配置
- `src/components`：复用组件
- `docs/API_CONTRACT.md`：前后端契约说明

前端启动方式：

```bash
cd front
npm install
npm run dev
```

## 3. 后端结构

后端目录中最重要的部分是：

- `src/main/java/com/nba/backend/controller`：HTTP 接口
- `src/main/java/com/nba/backend/service`：业务逻辑
- `src/main/java/com/nba/backend/repository`：JPA 数据访问
- `src/main/java/com/nba/backend/entity`：实体模型
- `src/main/resources/application.yml`：默认配置
- `scripts/sql`：数据库维护脚本
- `scripts/spark_nba_ml.py`：Spark 预测脚本

后端启动方式：

```bash
cd backend-spring
mvn spring-boot:run
```

默认数据源指向 `master:3306/nba_stats`，因此在本地环境中如果没有这个主机名，需要先通过环境变量覆盖数据库连接。

## 4. 数据流向

1. 用户在前端页面操作
2. 前端通过 `/api` 调用后端
3. 后端通过 JPA 读取和写入 MySQL / MariaDB
4. Spark 预测脚本从 HDFS 读取比赛数据并写回预测快照
5. SQL 脚本负责索引、回填和维度补齐

## 5. 维护顺序建议

如果你要从干净环境重新恢复项目，建议按下面顺序检查：

1. 确认数据库 schema 已初始化
2. 导入基础数据
3. 执行索引脚本
4. 执行高级指标回填脚本
5. 执行维度补齐脚本
6. 启动后端并访问 API
7. 启动前端并检查核心页面
